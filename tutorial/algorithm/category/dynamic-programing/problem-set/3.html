    背包问题入门

    给定一个背包重量上限 limit，给定n种物品的价值 value[n] 及其对应重量 weight[n]，希望在重量约束下使得物品价值最大化。

    0-1 背包

    每种物品只有一个，可以选择放入或者不放。此时状态转移函数较简单，对每个物品分拿或者不拿，且拿不拿当前物品可以达到的最大价值可以由之前物品来决定，此时：

    • 拿：dp[i][j] = dp[i-1][j-weight[i]] + value[i] （需要容量有效，即 j-weight[i] >= 0）
    • 不拿：dp[i][j] = dp[i-1][j]
    class Solution:
    def packProblem1(self, weight: List[int], value: List[int], limit: int) -> int:
        dp = [[0] * (limit + 1) for _ in range(len(weight))]
 
        # dp[N+1][W+1]
        # dp[i][j] 表示第 i 个物品和容量为 j 的最大值
        # dp[0][:] = 0
        # dp[:][0] = 0
 
        for i in range(len(weight)):  # 对第 i 个物品
            for j in range(limit + 1):  # 容量限制是 j 时, range(limit+1) 的最后一个循环 j = limit
                dp[i][j] = max(
                    dp[i - 1][j],  # 不拿这个物品
                    (dp[i - 1][j - weight[i]] + value[i]) if j - weight[i] >= 0 else 0  # 拿这个物品
                )
 
        return dp[-1][-1]  # dp[N-1][W]

    以 weight=[1, 3, 4], value=[15, 20, 30], limit=4 为例，算法运行过程中 dp 的更新过程为：

    完全背包

    每种物品有无限多个，即一个物品可以重复拿，会发现此时只需要把拿的状态转移函数从 dp[i-1][j-weight[i]] + value[i] 改为dp[i][j-weight[i]] + value[i]稍加修改即可，此时：

    • 拿：dp[i][j] = dp[i][j-weight[i]] + value[i] （需要容量有效，即 j-weight[i] >= 0）
    • 不拿：dp[i][j] = dp[i-1][j]
    class Solution:
    def packProblem2(self, weight: List[int], value: List[int], limit: int) -> int:
        dp = [[0] * (limit + 1) for _ in range(len(weight))]
 
        for i in range(len(weight)):  # 对第 i 个物品
            for j in range(limit + 1):  # 容量限制是 j 时, range(limit+1) 的最后一个循环 j = limit
                dp[i][j] = max(
                    dp[i - 1][j],  # 不拿这个物品
                    (dp[i][j - weight[i]] + value[i]) if j - weight[i] >= 0 else 0  # 拿这个物品
                )
        return dp[-1][-1]  # dp[N][W]

    以 weight=[1, 3, 4], value=[15, 50, 30], limit=6 为例，算法运行过程中 dp 的更新过程为：

    和 01 背包的过程对比，可以用来理解 i-1 到 i 的过程的本质。

    多重背包

    介于多重背包和完全背包之间，此时每个物品的个数是有限的。可以理解为 01 背包和完全背包是多重背包的特殊形式，多重背包是 01 和完全的通解。

    完全背包中，物品也可以拥有上限数量，只需要令 count[i] = limit//weight[i] 即可

    作为通解，复杂度在不额外引入优化数据结构（九讲中提到的单调队列）的情况下，没有办法达到 $O(NW)$，最多达到 $O(O(W\sum^N_i(log_2K_i)))$

    和完全背包不同，多重背包中的物品是有限的，因此第 i 个物品可以被展开为 count[i] 个等价值等重量的物品，从而将问题转化为 01 背包问题：

    class Solution:
    def packProblem3(self, weight: List[int], value: List[int], count: List[int], limit: int) -> int:
        # 展开物品
        for i in range(len(count)):
            for _ in range(count[i]): # 展开
                weight.append(weight[i])
                value.append(value[i])
 
        # 01 背包
        return self.packProblem1(weight, value, limit)

    这种展开也可以理解为，对第 i 个物品，决定拿 k 个还是不拿 k 个，同样也是 01 背包的问题：

    class Solution:
    def packProblem3(self, weight: List[int], value: List[int], count: List[int], limit: int) -> int:
        dp = [[0] * (limit + 1) for _ in range(len(weight))]
 
        for i in range(len(weight)):  # 对第 i 个物品
            for j in range(limit + 1):  # 容量限制是 j 时, range(limit+1) 的最后一个循环 j = limit
                for k in range(count[i] + 1): # 第 i 个物品一次拿 k 个拿不拿？
                    nweight = j - k * weight[i]  # 容量 j 时一次需要拿 k 个物品还剩的容量
                    if nweight < 0:
                        break
                    dp[i][j] = max(
                        dp[i - 1][j],  # 不拿这个物品
                        (dp[i - 1][nweight] + k * value[i]) if nweight >= 0 else 0  # 拿 k 个 i 物品
                    )
        return dp[-1][-1]  # dp[N-1][W]

    该算法复杂度为 $O(W\sum^{N}_{i}C_i)$，其中 $C_i$ 的数量会显著影响到算法开销。

    对数级优化

    在展开物品时，令 k = limit // weight[i]，用二进制表示 k，则二进制的位数 kb 为 $log_2(k)$，令 kb 二进制串中第 j 位的物品的价值 value_k[j] = value[i]*2^j，重量 weight_k[j] = weight[i]*2^j。此时对二进制位的组合可以模拟所有 k 个物品的选择情况，且二进制位数 kb << k，可以实现开销更小的展开，即开头所说的最优复杂度。

    class Solution:
    def packProblem3(self, weight: List[int], value: List[int], count: List[int], limit: int) -> int:
        for i in range(len(count)):
            k = min(limit // weight[i], count[i])
            j = 1
            while j <= k:
                weight.append(weight[i] * j)
                value.append(value[i] * j)
                j *= 2
 
        # 01 背包
        return self.packProblem1(weight, value, limit)

    对数级展开过程和背包过程结合

    当 count[i] * weight[i] > limit 时，第 i 个物品可以看成是完全背包中的情况（不可能不够用）；当 count[i] * weight[i] <= limit 时，可以按照对数级优化进行二进制展开。这两种情况可以合并在一起，从而省略了对原数组的操作

    class Solution:
    def packProblem3(self, weight: List[int], value: List[int], count: List[int], limit: int) -> int:
        dp = [[0] * (limit + 1) for _ in range(len(weight))]
 
        for i in range(len(weight)):  # 对第 i 个物品
            for j in range(limit + 1):  # 容量限制是 j 时, range(limit+1) 的最后一个循环 j = limit
                if count[i] * weight[i] >= limit:
                    # 完全背包
                    dp[i][j] = max(
                        dp[i - 1][j],  # 不拿
                        dp[i][j - weight[i]] + value[i] if j - weight[i] >= 0 else 0  # 拿
                    )
                else:
                    # 01 背包，分 log(count[i]) 次选择
                    k = 1  # k = 2^b
                    while k < count[i]:
                        dp[i][j] = max(
                            dp[i - 1][j],  # 不拿
                            dp[i - 1][j - weight[i] * k] + value[i] * k if j - weight[i] * k >= 0 else 0
                        )
                        k *= 2
 
        return dp[-1][-1]  # dp[N-1][W]

    该代码可以通用与混合背包问题（九讲第四讲）。

    分组背包

    分组背包问题中，物品被分作 N 个组，每组中的物品互相冲突。该题目可以看成是 01 背包的变种，题目由每个物品拿或不拿变成了每个组拿或不拿，拿的话拿哪个物品，所以每次比较第 k 组的第 i 个物品时，要比较三个状态：

    • dp[k-1][j] ：不拿第 k 组的物品
    • dp[k][j] ：拿第 k 组之前的物品
    • dp[k-1][j-weight[i]]+value[i]：拿新物品
    class Solution:
    def packProblem4(self, weight: List[int], value: List[int], group: List[List[int]], limit: int) -> int:
        dp = [[0] * (limit + 1) for _ in range(len(group))]
 
        for k, g in enumerate(group):
            for j in range(limit + 1):
                for i in g:  # g 组的第 i 个物品
                    dp[k][j] = max(
                        dp[k - 1][j],  # 第k 组什么都不拿
                        dp[k][j],  # 保持原来选择的物品不变
                        dp[k-1][j - weight[i]] + value[i] if j - weight[i] >= 0 else 0,  # 拿第 k 组第 i 个物品
                    )
 
        return dp[-1][-1]  # dp[N-1][W]

    卡牌游戏能达到的最大攻击力

    一个卡牌游戏，游戏中初始攻击力为 1，有两种卡牌，使用祝福每次攻击力 + 1，使用血怒让当前攻击力减半（向下取整），但让下次祝福的攻击力增益多 1 点。

    现在，按顺序给定一个卡牌序列，序列里 0 表示祝福，1 表示血怒。对每张卡牌可以选择是否使用。求最终能达到的最大攻击力。序列长度在 [1,10000]。

    令 mem[j] 为 i 价值时存储的最大攻击力数值，每个卡牌遍历更新一遍 mem。复杂度 $O(N^2)$

    class Solution:
    def packProblemGeneral(self, nums: List[int]):
        n = len(nums)
        mem = defaultdict(int)
        mem[0] = 1
        ans = 0
        for i in range(n):
            for k, v in list(mem.items()):
                if nums[i] == 0:
                    mem[k + v] = max(mem[k + v], v)
                    ans = max(k + v, ans)
                else:
                    mem[k // 2] = max(mem[k // 2], v + 1)
        return ans

    背包问题优化

    空间复杂度优化

    背包问题的初始算法空间复杂度为 $O(NW)$，一般通过观察状态转移公式或算法代码可以将初始分配空间进行压缩为$O(W)$，这里 W 是背包容量 limit。

    01 背包

    因为 dp[i][j] 依赖于 dp[i-1][j] 和 dp[i-1][j - weight[i]] ，即对 f[j] 和 f[j - weight[i] < j] 的要求是上一次外循环的，所以可以通过逆序更新 j 到 dp[i][j] 节省 dp[i-1] 的数组空间。

    class Solution:
    def packProblem1(self, weight: List[int], value: List[int], limit: int) -> int:
        dp = [0] * (limit + 1)
 
        for i in range(len(weight)):  # 对第 i 个物品
            for j in reversed(range(limit + 1)):  # 容量限制是 j 时, range(limit+1) 的最后一个循环 j = limit
                dp[j] = max(
                    dp[j],  # 不拿这个物品
                    (dp[j - weight[i]] + value[i]) if j - weight[i] >= 0 else 0  # 拿这个物品
                )
 
        return dp[-1]  # dp[N-1][W]

    完全背包

    因为 dp[i][j] 依赖于上一外循环的 dp[i-1][j] 和当前循环的 dp[i][j - weight[i]] ，上一循环的 dp[i-1][j] 可以直接被压缩，当前循环的 dp[i][j - weight[i]] 要求 j 顺序遍历

    class Solution:
    def packProblem2(self, weight: List[int], value: List[int], limit: int) -> int:
        dp = [0] * (limit + 1)
 
        for i in range(len(weight)):  # 对第 i 个物品
            for j in range(limit + 1):  # 容量限制是 j 时, range(limit+1) 的最后一个循环 j = limit
                dp[i][j] = max(
                    dp[j],  # 不拿这个物品
                    (dp[j - weight[i]] + value[i]) if j - weight[i] >= 0 else 0  # 拿这个物品
                )
        return dp[-1]  # dp[N][W]

    多重背包和分组背包

    根据上面的思路，会发现多重背包同时依赖了 j 两侧的状态，没有办法直接压缩。但注意到完全背包的分支可以转换到 01 背包，因此在转换后，可以按照 01 背包的方式压缩。

    class Solution:
    def packProblem3(self, weight: List[int], value: List[int], count: List[int], limit: int) -> int:
        dp = [0] * (limit + 1)
 
        for i in range(len(weight)):  # 对第 i 个物品
            for j in reversed(range(limit + 1)):  # 容量限制是 j 时, range(limit+1) 的最后一个循环 j = limit
                max_v = min(count[i], limit//weight[i])
                # 01 背包，分 log(count[i]) 次选择
                k = 1  # k = 2^b
                while k < max_v:
                    dp[i][j] = max(
                        dp[j],  # 不拿
                        dp[j - weight[i] * k] + value[i] * k if j - weight[i] * k >= 0 else 0
                    )
                    k *= 2
 
        return dp[-1]  # dp[N-1][W]

    分组背包的优化思路和 01 背包完全相同，依赖的是上一次循环的左侧，所以是逆序遍历。唯一注意的是之前代码中的 max(dp[k-1][j], dp[k][j], ...) 合并为 max(dp[j], ...) 的过程没有问题，不会导致错误比较。

    class Solution:
    def packProblem4(self, weight: List[int], value: List[int], group: List[List[int]], limit: int) -> int:
        dp = [0] * (limit + 1)
 
        for k, g in enumerate(group):
            for j in reversed(range(limit + 1)):
                for i in g:  # g 组的第 i 个物品
                    dp[j] = max(
                        dp[j],  # 第 k 组什么都不拿
                        dp[j - weight[i]] + value[i] if j - weight[i] >= 0 else 0,  # 拿第 k 组第 i 个物品
                    )
 
        return dp[-1]  # dp[N-1][W]

    时间复杂度优化

    内外循环置换

    在背包容量范围内物品的最大价值问题也可以转化为最大价值问题，找到某个价值下的最小重量。此时复杂度由 $O(NW)$ 转换为 $O(NV)$。

    和重量不同，不同物品组合出的价值数量是有限的，此时用 dict 会更好。在最差情况下，算法退化为 $O(2^N)$

    class Solution:
    def packProblem1(self, weight: List[int], value: List[int], limit: int) -> int:
        dp = defaultdict(lambda: defaultdict(lambda: float('inf')))
 
        # trick，表示什么都不拿
        dp[-1][0] = 0
 
        max_v = 0
        for i in range(len(value)):  # 对第 i 个物品
            for k, w in dp[i - 1].items():  # 遍历前 i-1 能达到的所有可能的价值
                dp[i][k] = min(w, dp[i][k])  # 不拿
                max_v = max(k, max_v)
                if w + weight[i] <= limit:  # 拿，注意检查重量边界
                    dp[i][k + value[i]] = min(dp[i][k + value[i]], w + weight[i])
                    max_v = max(k + value[i], max_v)
 
        return max_v

    常数级优化*

    在《九讲》中，提到了针对 limit 层循环的一个常数级优化。以 01 背包问题为例。

    • 在有空间复杂度优化的情况下，遍历到第 i 个物品时，对 limit 的内循环可以省略 j = [0,weight[i])，因为此时 j - weight[i] < 0，dp[i][j] = dp[i-1][j]。
    • 第 n-1 个物品（最后一个物品）的最后一个循环（j==limit）依赖的拿的状态是 dp[n-2][limit-weight[n-1]]，要得到第 dp[n-2][limit-weight[n-1]]（算法的循环执行到 i=n-2, j=limit-weight[n-1]） 则要求 dp[i-1][j-weight[i]] 即 dp[n-3][limit-weight[n-1]-weight[n-2]]...推导可得第 i 个物品至少要求得的状态是 dp[i][limit-sum(weight[j+1:])]
    • 因为不拿的最终状态是 dp[n-1][limit] ，所以最终第 i 个物品的内循环次数是 range(b, limit+1)，其中 b = max(weight[i], limit - sum(weight[i+1:]))，即上面两种可以省略的情况取并集，表现为 max 操作。
    class Solution:
    def packProblem1(self, weight: List[int], value: List[int], limit: int) -> int:
        dp = [0] * (limit + 1)
 
        for i in range(len(weight)):  # 对第 i 个物品
            b = max(weight[i], limit - sum(weight[i+1:]))
            for j in range(b, limit + 1):  # 容量限制是 j 时, range(limit+1) 的最后一个循环 j = limit
                dp[j] = max(
                    dp[j],  # 不拿这个物品
                    (dp[j - weight[i]] + value[i]) if j - weight[i] >= 0 else 0  # 拿这个物品
                )
 
        return dp[-1][-1]  # dp[N-1][W]

    原文提到，该优化在 limit 较大时作用较显著。
