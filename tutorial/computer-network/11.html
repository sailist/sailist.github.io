    5.总结与辨析

    拥塞/流量控制

    拥塞的具体原因包括：

    • 节点存储容量（缓冲区）不够
    • 处理机速度太低
    • 线路容量（带宽）不够

    在不同的模型层，都需要在一定程度上解决该问题

    • 在数据链路层，实现的是点对点的通信，双方缓冲区的大小根据滑动窗口协议而定
    • 传输层实现的是端到端的通信，某一时刻，一台主机可能同时与多台主机建立了连接，多个连接必须有多组缓冲区，所以缓冲区的动态分配和管理策略与数据链路层相比较要复杂得多
    • 网络层直接面临拥塞（和链路层、传输层只在数据两端感受拥塞的结果不同），该层的拥塞控制由慢（预防性）到快（反应性）可以采用以下策略：
      • 网络供给
      • 流量感知路由
      • 准入控制：不再建立新的虚电路（虚电路：一条端到端的连接）
      • 流量限制：
      • 负载脱落：直接丢包

    差错控制

    https://blog.csdn.net/gao1440156051/article/details/9122207 (opens in a new tab)

    为了避免数据的传输不出错，会：

    • 通过带序确认和超时重传保证数据传递有序、不丢失、不重复
    • 通过校验和保证数据不出错。

    在链路层、网络层、传输层中需要差错控制。

    • 链路层保证帧的可靠性，通过循环冗余检验 CRC 进行差错校验，如果出错就丢包
    • 网络层保证 IP 协议中头部的可靠性，通过头部校验和保证 IP 数据报无误（比特的准确性由链路层保证）
    • 传输层保证通过观察数据是否未收到，对底层进行确认和重传两个操作（只针对 TCP、UDP 不考虑重传）

    单播&组播&广播

    单播

    • 链路层上的单播发生在任意两个有物理连接的设备（假设为 A - B）之间，在设备 A 上封帧，帧头包含源地址（A 设备 MAC 地址）和目的地址（B 设备 MAC 地址），即可将帧发送到 B 设备。
    • 网络层上的单播发生在任意两个有逻辑连接的设备之间。假设连接为 A-B-C，从 A 发送 IP 数据包到 C，会依次经过
      • DNS 解析（根据域名解析 C 的 IP 地址，如果已经得知，可以忽略）
      • 建立连接（TCP 协议要求，UDP 可略）
      • 发送 IP 数据报，数据报通过下层链路层，ARP 协议、路由算法，经过多跳发送到目标设备（C）上。

    广播

    • 链路层上的广播通过将目的 MAC 地址设置为全 1 实现。因为路由器不转发目的地址是广播地址的数据报，所以链路层广播只发生在本地网络（局域网）中。
    • 网络层上的广播通过将主机号对应的位设置全 1 实现。虽然是在网络层上广播，但是一般情况下广播只会发生在本地网络中，所以也称网络层上的广播为受限广播。

    ? 虽然网络层上也可以设置广播，但广播的 IP 数据报一般也只是在局域网内传递，在网关处会被阻拦（除非路由器做了其他判断）。

    参考 https://www.zhihu.com/question/20643350/answer/112565403 (opens in a new tab), https://www.zhihu.com/question/20643350/answer/112565403 (opens in a new tab)

    一次 HTTP 请求的过程

    客户端视角：

    • 域名 DNS 解析
      • 依次从浏览器缓存、本地 DNS 服务器缓存、DNS 服务器中取 IP
    • 建立 TCP 连接，从服务器接收数据
    • 发送/接收无连接的 UDP 数据报
    • 浏览器解析数据，渲染页面

    服务端视角：

    • 接受来自客户端的请求
    • （可选）负载均衡
    • 请求资源判断是否有权限，返回相应条件下的结果（和在服务端进行各种处理）

    操作系统视角：

    • 接收来自浏览器的软中断，执行系统调用，建立 socket 返回给上层应用
    • 通过 socket 和目标机器（服务器）进行 IO
      • input: 当网卡产生硬中断时，从网卡接收数据
      • output: 从 socket 获取的数据写入到网卡中

    连接 wifi 的过程发生了什么

    ARP 表、路由表、Mac 表

    • 路由表建立了当前设备中目的 IP 地址到下一跳地址的映射，用于网络层在网络间的转发，需要配合 ARP 表的使用
    • ARP 表建立了 IP 地址到 MAC 地址的映射，用于网络层向下注入到链路层
    • Mac 表建立了 Mac 地址到端口的映射，用于链路层内局域网下的转发
