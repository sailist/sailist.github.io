    内存管理的软硬件支持

    CPU-内存架构

    如果每个处理器核心共享相同的内存地址空间，这样的架构就称为UMA(Uniform Memory Access)。

    对非一致性内存访问（NUMA，Non-Uniform Memory Access）不同的内存器件和 CPU 核心从属不同的 Node，每个 Node 都有自己的集成内存控制器（IMC，Integrated Memory Controller）。

    在 NUMA 架构下，CPU 访问不同的物理内存的速度会存在十分明显的差别。这是 Linux 内核内存管理的硬件基础。

    MMU

    现代的内存管理方案是硬件和软件（操作系统）共同协同后得到的。在硬件上提供的与内存有关的支持包括三方面：

    • **页表支持：**提供虚拟地址到物理地址转换的硬件支持，包括存储页表基地址的寄存器（如 TTBRx_ELx、CR[0-3] 寄存器）、页表访存单元（table work unit，用于访问在物理内存中的页表的操作）等
    • 缓存： 在寻址时能够更快的实现虚拟地址到物理地址的转换，这包括 TLB、paging structure caches、高速缓存等
    • 缺页中断：在无法正确将虚拟地址转换为物理地址时，向处理器发出缺页硬件中断

    这些功能主要是由 MMU 提供的。

    一些 CPU（如 i386）中还额外提供了段式存储管理，即程序中所使用的地址一律先进行段式映射（段基址:段偏移），再进行页式映射，这一操作在硬件层面上无法绕过，但是 Linux 在软件层面上将全局段描述符表 GDT 中用到的段描述项均设置为基地址为 0 的 4GB 大小的虚拟空间，仅在内核态和用户态标识上存在区分。因此可以理解为 Linux 内核并不是所谓的段页式内存管理，而是在 hack 式的绕过了硬件的段式映射要求后，使用的页式内存管理。

    在硬件的约束下，Linux 内核设计了不同的数据结构和接口，这包括：

    • 数据结构
      • struct page：每个 page 数据结构代表着一个物理页面，包含全部物理页面的 page 数组由一个全局指针变量 mem_map 定位。
      • struct mm_struct：内存描述符，每个进程分配一个 mm_struct，此外内核有一个单独设置的 mm_struct。mm_struct 表明了对虚存空间的需求
      • struct vm_area_struct：每个 vm_area_struct 代表虚拟内存空间中一段连续的、具有相同访问权限的区域，每个进程有多个 vm_area_struct，用链表或平衡树（AVL 树/红黑树）来组织。
      • struct vm_struct：和 vm_area_struct 类似，不过 vm_area_struct 是给用户进程用的，而 vm_struct 是给内核使用的
      • linux/mmzone.h -> struct zone：描述物理页面的 page 数组被划分为 ZONE_DMA、ZONE_NORMAL、ZONE_HIGHMEM（ZONE_HIGHMEM 为高端内存，可能有）三个管理区，用于不同的内存需求（如 ZONE_DMA 用于专供 DMA 的 IO 操作）。
      • linux/mmzone.h -> struct pglist_data：即 pg_data_t，每个 pg_data_t 管理多个 zone
      • struct pglist_data *node_data[MAX_NUMNODES]：用于 NUMA 结构，每个 CPU node 分配一个 pglist_data。
      • ...
    • 操作模块
      • buddy 分配器：以页为单位分配内存，每个 zone 会分配一个 buddy 内存分配。
      • slab 分配器（kmalloc、kfree）：用于内核态细粒度空间分配
      • malloc、free：用于用户态堆内存细粒度分配
      • brk、sbrk：扩展堆边界【？存疑】
      • mmap、munmap：建立虚拟内存和文件之间的映射，这里的文件，可以是硬盘上的具体的某个文件，也可以是物理内存上的一段空间（匿名文件/匿名映射）。vm_area_struct 实际是在 mmap 执行过程中创建的。
      • 缺页中断服务程序 do_page_fault：页错误处理，会根据不同的缺页类型调用不同的缺页函数，包括抛出异常、分配物理页面、将硬盘换入物理内存（ext2_readpage()）等
      • 对内存回收的相关接口
      • ...

    内存管理大图

    Linux 实操

    Linux 系统下查看内存管理区可以通过 cat /proc/buddyinfo 来完成：

    cat /proc/buddyinfo
> Node 0, zone      DMA      1 ...
> Node 0, zone    DMA32    975 ...
> Node 0, zone   Normal     71 ...

    寻址长度可以在 cat /proc/cpuinfo 中找到

    cat /proc/cpuinfo | grep address
address sizes	: 48 bits physical, 48 bits virtual
